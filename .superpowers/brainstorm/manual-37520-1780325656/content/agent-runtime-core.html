AgentRuntime First

第一版先跑通单个 agent runtime，把 memory、log、trace、MCP、skills 做成可挂载服务。

Runtime Core Boundary
AgentRuntime Core: lifecycle, loop, step control, stop conditions
Memory Service
Log Service
Trace Service
Skill Registry
MCP Registry
Tool Registry
Execution Adapter: local shell / coding CLI / model provider
Event Stream: runtime.started, step, tool.call, trace, log, completed

这个边界的好处

  • 单 agent 可以独立测试和运行
  • 多 agent 只是多个 runtime 的编排
  • Workbench 只订阅事件流，不侵入核心
  • memory/log/trace 可以先内存实现，后续替换成 DB/OTel

第一版先不做

  • 复杂多 agent 调度
  • 完整消息通道矩阵
  • 复杂权限系统
  • 跨机器 worker 池
