Spec 如何承载演进？

关键问题：最终事实和讨论过程要不要放在同一个文件里。

A

Single Spec With History

spec.md 同时放最终设计和完整讨论/变更记录。

B

Current Truth + Decision Log

spec.md 只保留当前最终事实；decisions.md 记录讨论、方案、tradeoff 和为什么改。

C

RFC Snapshots

每轮重要讨论都生成一个 RFC 快照，最终 spec 从被接受的 RFC 汇总而来。

推荐: B
docs/modules/agent-runtime-core/
spec.md # 当前事实: 最终读这个就能实现
decisions.md # 过程记录: 场景、方案、tradeoff、修订原因
tasks.md # 当前计划: 按什么顺序实现
checklist.md # 验收标准: 可观察、可执行
