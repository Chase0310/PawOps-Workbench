第一版范围怎么切？

先决定 MVP 的重心，后面的架构边界、存储、消息通道和 UI 都会跟着这个选择变化。

A

Agent Runtime First

先做本地多 agent 编排内核、agent/skill/MCP 配置模型、CLI 进程适配和事件流；UI 只做最小可观测面板。

B

Workbench First

先做可用工作台：agent 列表、会话、任务看板、终端视图、消息流；底层 runtime 保持够用但不追求插件化完备。

C

Integration Hub First

先做 CLI 工具和消息通道接入层，让 Codex、Claude Code、Gemini CLI、Slack/Discord/飞书等能统一进出。

建议的长期分层
Workbench UI / Channels
Orchestrator: tasks, sessions, routing, approvals
Agent Runtime: agent profiles, skills, MCP, memory, permissions
Adapters: Coding CLIs, shells, repos, issue trackers, chat platforms
