技术选型: 左侧栏先以 Agent 为中心还是以 Run 为中心

工作台已经确定是三栏结构。现在决定左侧主导航的第一版信息组织方式。

A

Agent-centric

先看有哪些狗狗 agent，再进入该 agent 下的 runs。适合强调角色和能力差异。

B

Run-centric

先看所有 runs，再回看它们属于哪个 agent。适合调试和任务跟踪。

C

Split Sections

左侧同时有 Agents 和 Runs 两个区块，互相关联但不强制层级。兼顾两者，适合第一版工作台。

推荐: C
Agents

- Builder Dog
- Reviewer Dog
- Ops Dog

Active Runs

- run-001 (Builder Dog)
- run-002 (Reviewer Dog)

Recent Runs

- run-000

Why C fits

  • 你做的是多狗狗 agent 工作台，agent 视角不能丢。
  • 调试时用户通常又更关心当前和最近的 runs。
  • 比纯层级结构更灵活，也更符合第一版工作流。

Accepted cost

  • 左侧信息设计稍微复杂一点。
  • 要控制列表密度，避免左栏变成噪音区。
