技术选型: Workbench 第一版最小交互动作

三栏布局和主视图都收敛了。现在决定第一版工作台用户可以直接执行哪些动作。

A

Read-only Monitor

只能看 agent、run、事件和详情，所有操作仍通过外部 CLI 或开发接口进行。实现最轻，但工作台价值有限。

B

Essential Runtime Actions

支持启动 run、提交输入、取消 run、切换 agent、查看历史。第一版已具备基本闭环。

C

Full Control Console

一开始就支持 agent 编辑、profile 管理、权限修改、预算调整、trace 导出等完整控制能力。明显过重。

推荐: B
Start run with selected agent
View active and recent runs
Submit structured input when waiting-for-input
Cancel run
Switch between agents and runs

Why B fits

  • 工作台已经能真正驱动 runtime，而不只是观测。
  • 和前面定义的 Method API、structured input、cancel 模型完全对应。
  • 足以验证桌面端和 Web 调试端的主要交互闭环。

Accepted cost

  • 要做最小表单和动作反馈，而不只是订阅事件。
  • 第一版仍然不做 profile 编辑和高级配置流。
