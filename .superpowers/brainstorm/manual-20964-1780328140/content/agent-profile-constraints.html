技术选型: Permissions 与 Budget 的第一版粒度

Profile 已经确定会带 permissions 和 budget。现在决定这两个字段第一版要建模到多细。

A

Coarse-grained

只做粗粒度开关，例如 allowShell、allowWrite、allowNetwork、maxRuntimeMinutes。最容易跑通。

B

Balanced Constraints

权限按能力域划分，预算按时间、步骤、token 或成本上限建模。适合第一版稳定落地。

C

Fine-grained Policy

一开始就支持路径级、命令级、工具级、网络目标级白名单，以及复杂配额策略。

推荐: B
permissions:
shell: none | read-only | full
filesystem: read-only | workspace-write
network: blocked | allowed
mcp: allowedServerIds[]
budget:
maxRunSeconds
maxSteps
maxCostUsd? or maxTokens?

Why B fits

  • 比粗粒度更真实，已经能限制危险能力。
  • 比细粒度策略轻很多，不会把第一版拖进策略系统。
  • 和 CLI executor、MCP server、runtime lifecycle 都能对齐。

Accepted cost

  • 仍然需要 runtime 和 executor 一起遵守这些约束。
  • 路径级白名单、命令级策略等高级能力要后置。
