技术选型: Coding CLI Adapter 第一版支持范围

Runtime 和 Agent Profile 主骨架已基本收敛。现在决定第一版 CLI 适配层先支持哪些执行器。

A

Single CLI First

第一版只支持一个 coding CLI，最快跑通，但平台价值和抽象验证都有限。

B

Small Curated Set

第一版支持 2-3 个最典型的 coding CLI，验证统一适配接口是否成立。

C

Open Adapter Market

一开始就面向任意 CLI 开放注册。扩展性最强，但第一版边界会非常松散。

推荐: B
v1 adapter targets: Codex CLI, Claude Code, Gemini CLI
all adapters implement the same executor-facing contract
focus on launch, stream output, waiting-for-input, interrupt, exit semantics
defer long-tail CLIs until the base contract is proven

Why B fits

  • 已经能验证“统一 executor 接口”是否站得住。
  • 比单一 CLI 更能暴露适配层真正的共性和差异。
  • 还能把第一版范围控制在可落地区间。

Accepted cost

  • 要处理 2-3 种 CLI 的行为差异。
  • 第一版不追求通用插件市场或任意命令兼容。
