技术选型: Agent Profile 发现与加载

Profile 格式已基本收敛。现在决定 runtime 如何找到、校验并注册这些 agent。

A

Filesystem Scan

启动时扫描约定目录，读取 manifest 并注册。最直观，最适合本地桌面和开发场景。

B

Explicit Registration

调用方显式传入 profile 列表或 registry。控制力强，但扩展和插件体验较差。

C

Hybrid Loader

runtime 既支持扫描目录，也支持调用方直接注册 profile。兼顾桌面体验和嵌入式集成。

推荐: C
desktop mode: scan local profiles directory
tests or embedded mode: register profiles programmatically
all profiles pass schema validation before registration
runtime builds a stable profile registry from validated manifests

Why C fits

  • 本地目录管理 agent 很自然，适合桌面产品。
  • 测试和嵌入模式不必依赖文件系统扫描。
  • 比纯扫描更容易控制加载顺序和注入测试 profile。

Accepted cost

  • 要维护两种入口，但底层注册逻辑必须保持同一套。
  • 需要定义扫描目录、覆盖规则和冲突处理。
