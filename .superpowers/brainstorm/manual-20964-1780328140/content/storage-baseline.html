下一层: Memory / Log / Trace 存储基线

Runtime core 不直接绑定存储，但第一版需要一个默认落地方式。

A

In-memory First

最快实现，适合测试 runtime core，但重启后丢失，不适合桌面产品。

B

SQLite Local Store

本地优先、可分发、能记录 runs/events/logs/traces，适合桌面端和 Web 调试。

C

Postgres First

适合多人和服务端部署，但第一版本地桌面分发会更重。

推荐: B + Test Adapter
runtime-core: depends on Store interfaces only
runtime-testing: in-memory store for tests
runtime-sqlite: local durable store for desktop/debug
later: Postgres adapter for hosted/multi-user mode
