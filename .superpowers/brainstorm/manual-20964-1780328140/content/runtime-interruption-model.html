技术选型: 中断与取消模型

我们已经支持 pause、cancel、waiting-for-input。现在决定 runtime 如何中断正在运行的 executor。

A

Hard Kill Only

取消时直接终止执行进程。实现简单，但会牺牲交互性和安全退出。

B

Soft Interrupt, Then Hard Kill

先请求优雅中断，超时后再强制终止。适合 CLI executor 的真实行为。

C

Executor-specific Policy

每种 executor 自己定义 pause/cancel 行为，runtime 不统一约束。灵活但会破坏一致性。

推荐: B
runtime.cancel(runId)
executor receives soft interrupt request
runtime emits cancelling event
if executor exits in time -> cancelled
if timeout expires -> hard kill and cancelled

Why B fits

  • 更贴近真实 coding CLI 的中断方式。
  • 避免一上来就粗暴 kill 进程。
  • runtime 仍然保留最后兜底能力。

Accepted cost

  • 要定义 soft interrupt 的接口和超时策略。
  • 不同 CLI 对优雅退出支持程度不同，需要适配层处理。
