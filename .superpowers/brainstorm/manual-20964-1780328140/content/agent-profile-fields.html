技术选型: Agent Profile 第一版字段边界

Profile 的发现方式已经定了。现在决定 manifest 第一版必须包含哪些字段。

A

Minimal Fields

只保留最小集合：id、name、executor、prompt、skills。最快，但很快会不够。

B

Balanced Fields

包含身份、执行器、提示、skills、MCP、权限、预算和标签。适合作为第一版稳定骨架。

C

Rich Fields

一开始就加 memory policy、trace policy、tool overrides、hooks、routing rules、channel policy。

推荐: B
id
name
role
description
executor
prompt
skills[]
mcpServers[]
permissions
budget
tags[]
disabled?: boolean

Why B fits

  • 已经能表达大多数狗狗 agent 的稳定身份和能力边界。
  • 不会太轻，避免很快回头改 schema。
  • 也不会太重，避免把所有运行时策略都塞进 profile。

Accepted cost

  • 有些高级策略先不进 profile，要延后到 runtime 或 hook 层。
  • 权限和预算字段要先定义一个足够克制的模型。
