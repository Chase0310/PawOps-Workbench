技术选型: Workbench 第一版最小视图集

前面的 runtime、profile、CLI adapter 主骨架已经基本清楚。现在决定第一版工作台最少必须呈现哪些界面。

A

Single Console View

只有一个运行控制台，展示当前 run 的事件和输出。实现最快，但无法很好管理多个 agent 和历史 run。

B

Three-pane Workbench

左侧 agent 与 run 列表，中间当前 run 事件流，右侧详情面板。第一版已经具备工作台形态。

C

Full Dashboard

一开始就做 agent 市场、trace 视图、配置页、历史分析等完整控制台。能力全，但明显超出第一版范围。

Why B fits

  • 已经能体现“工作台”而不只是日志窗口。
  • 足够支撑调试多 agent、多 run。
  • 和我们已经定义的事件模型、profile、run 生命周期天然对齐。

Accepted cost

  • 比单控制台多了一点 UI 结构成本。
  • 第一版仍然不做复杂历史分析、市场、配置中心。
