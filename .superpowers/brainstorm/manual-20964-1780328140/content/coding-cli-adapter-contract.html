技术选型: Codex / Claude Code 适配器统一接口

第一版只支持 Codex CLI 和 Claude Code CLI。现在决定它们共享的 adapter contract 长什么样。

A

Process Wrapper Only

适配器只暴露 spawn、stdin、stdout、kill。最薄，但 runtime 需要自己理解大量细节。

B

Session-oriented Adapter

适配器暴露 launch、sendInput、interrupt、dispose，并把 CLI 行为翻译成统一会话事件。

C

Full Conversation Engine

适配器自己维护大量对话语义和状态机，runtime 只接收高层结果。过早把复杂度推到适配层。

推荐: B
adapter.launch(context) -> session
session.subscribe(listener)
session.sendInput(input)
session.softInterrupt()
session.hardKill()
session.dispose()
normalized events:
started
output.chunk
input.requested
warning
completed
failed
interrupted

Why B fits

  • runtime 不必直接处理原始子进程细节。
  • Codex CLI 和 Claude Code 的差异可以在 adapter 内部消化。
  • 又不会让 adapter 演变成比 runtime 更重的系统。

Accepted cost

  • 需要先定义一套统一会话事件。
  • 两个 CLI 的输入/输出语义不一致时，要做保守归一化。
