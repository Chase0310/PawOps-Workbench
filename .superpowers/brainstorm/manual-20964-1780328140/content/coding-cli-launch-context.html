技术选型: CLI launch(context) 第一版上下文字段

适配器事件已经定了。现在决定 runtime 启动 Codex CLI 或 Claude Code 时，要统一提供哪些上下文。

A

Minimal Context

只传工作目录和初始输入。实现最简单，但很快不够表达权限、环境和 profile 差异。

B

Balanced Context

传入 runId、workdir、initialPrompt、env、profile summary、permissions、budget、metadata。

C

Rich Context

一开始就把 memory snapshot、trace config、tool registry、full structured policy 全塞进 launch context。

推荐: B
runId
workdir
initialPrompt
env
profileSummary
permissions
budget
metadata

Why B fits

  • 已经能表达 profile 与 runtime 对执行器的关键约束。
  • 不会让 adapter 自己回头查太多外部状态。
  • 比最小上下文更适合 Codex CLI 和 Claude Code 的真实使用。

Accepted cost

  • 需要明确哪些字段是 runtime 保证提供，哪些是可选。
  • 不能把 context 膨胀成“整个 run 世界快照”。
