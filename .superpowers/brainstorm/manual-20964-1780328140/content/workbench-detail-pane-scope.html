技术选型: 右侧详情面板第一版展示范围

左侧和中间主视角已经收敛。现在决定右侧详情面板第一版固定展示哪些信息。

A

Minimal Status Only

只显示当前状态和少量摘要。最轻，但不够支撑调试和理解 agent 边界。

B

Balanced Run Details

显示状态、当前 agent、executor、permissions、budget、input request、metadata 摘要。适合作为第一版工作台右栏。

C

Full Inspector

一开始就显示完整 profile、全部 metadata、trace、环境变量、原始事件。能力全，但信息噪声太大。

推荐: B
Run status: running / waiting-for-input / completed
Agent: Builder Dog
Executor: Codex CLI
Permissions: shell full, filesystem workspace-write, network blocked
Budget: 12 steps, 900s, $0.80
Current input request: approval needed
Metadata summary: CLI version, workdir, startedAt

Why B fits

  • 已经足够解释“这个 run 现在是什么、为什么这样跑”。
  • 和 profile、runtime、CLI adapter 的关键边界正好对齐。
  • 不会把右栏做成噪声很大的原始数据面板。

Accepted cost

  • 深度调试时仍然要从时间线或后续 trace 视图取更多细节。
  • 需要克制 metadata 展示，只给摘要，不给全量 dump。
