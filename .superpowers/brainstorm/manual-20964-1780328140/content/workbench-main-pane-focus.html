技术选型: 中间主面板以事件时间线为主还是以对话输出为主

左侧结构已经收敛。现在决定第一版中间面板的核心阅读模式。

A

Conversation / Output First

主面板先展示 agent 输出、CLI 内容和人机交互，对事件细节做弱化。更像聊天或终端。

B

Event Timeline First

主面板按时间线展示 run 的关键事件，输出内容作为时间线中的一种事件展开。

C

Dual Mode

第一版就提供“时间线 / 输出”双视图切换。更灵活，但 UI 和状态复杂度也更高。

推荐: B
09:41 runtime.started
09:41 step.started
09:42 output.chunk - Codex analyzing repository...
09:42 status.changed - waiting-for-input
09:43 input.requested - approval needed
09:44 output.chunk - resumed after input

Why B fits

  • 和我们前面定义的 typed events 完全对齐。
  • Workbench 第一版更偏调试与编排，不只是聊天界面。
  • 输出仍然能展示，但被放在更完整的运行上下文里。

Accepted cost

  • 比纯聊天输出界面对普通用户稍微技术化一些。
  • 要控制时间线密度，避免刷屏式噪声。
